Prompt engineering para executivos

Escrever um bom prompt não é decorar comandos secretos — é comunicar com clareza o que você quer, como faria com um assistente extremamente capaz, porém muito literal. Poucos princípios levam a maior parte do resultado. Este guia vai do essencial (que já muda tudo) ao controle de fluxos. Os exemplos são para copiar e testar.

Como usar este guia

  1. Comece pela anatomia do prompt, logo abaixo — é o molde que serve para quase tudo.
  2. Os quatro hábitos do Nível 1 já resolvem a maioria dos casos. Suba de nível quando precisar.
  3. Copie um exemplo, cole no ChatGPT ou Claude, e itere — a primeira resposta é sempre um rascunho.

A anatomia de um bom prompt

Um molde que serve para quase tudo. Você não precisa de todos os campos sempre — mas quanto mais completo, melhor o resultado.

Papel: aja como [especialista]. Contexto: [cole aqui o material ou descreva a situação]. Tarefa: [o que fazer], para [objetivo]. Formato: responda em [bullets / tabela / e-mail], até [X] palavras, tom [executivo]. Exemplos: siga o estilo de [exemplo, se houver]. Restrições: não [faça X]; se faltar informação, pergunte antes de assumir.
